# AIAAIC Repository — Data Cleaning & Exploration

Analysis pipeline for the [AIAAIC AI/Algorithmic Incident Repository](https://www.aiaaic.org/aiaaic-repository), a manually curated database of 2,243 AI incidents (2008–2026).

## Quick start

```bash
# 1. Install dependencies
pip install pandas matplotlib seaborn rapidfuzz

# 2. Place the raw CSV in this directory
#    File: AIAAIC_Repository_-_Incidents.csv

# 3. Clean the data
python clean.py

# 4. Explore and generate charts
python explore.py
```

## What `clean.py` does

1. **Parses the multi-row header** — the raw CSV has 3 header rows (title, columns, sub-columns) which pandas can't handle natively.
2. **Normalises taxonomy fields** — fixes ~50 known typos and near-duplicates (e.g. "Facial recgnition" → "Facial recognition", "Accountabiilty" → "Accountability").
3. **Deduplicates multi-coded values** — splits semicolon-delimited fields, dedupes, and rejoins.
4. **Adds derived columns**:
   - `year` — extracted integer year
   - `harm_breadth` — count of distinct harm tags across all harm fields
   - `has_consequence` / `has_response` — boolean flags
   - `detection_category` — maps news triggers into 6 high-level categories
   - `fairness_subtypes` — extracts protected characteristics from "Fairness - X" tags
   - `is_generative_ai`, `is_facial_recognition`, etc. — boolean tech flags
   - `has_harm_data`, `has_developer` — data quality flags
   - `is_partial_year` — flags 2026

## What `explore.py` produces

11 publication-quality PNG charts covering:
- Incident volume over time
- Technology regime change (GenAI takeover)
- GenAI share of incidents
- Ethical concern trends
- The accountability gap (incidents vs consequences)
- Detection method → consequence pipeline
- Developer risk profiles (heatmap)
- Developer accountability comparison
- Harm fingerprints by technology
- Field completeness (data quality)
- Fatal incidents trend

Plus summary CSVs and printed narrative observations.

## Known database limitations

Document these in any analysis using this data:

1. **Reporting lag** — 2025/2026 data is incomplete. Cross-reference with AIID.
2. **English-language bias** — 51% USA; non-English incidents are underrepresented.
3. **Harm field coding shift** — Individual vs societal harm coverage flipped around 2021; cross-year harm comparisons are unreliable.
4. **Declining response coverage** — response field dropped to 8% in 2024.
5. **Taxonomy fragmentation** — 241 technology values, 120 ethics values, many one-offs. The normalisation in `clean.py` handles known issues but isn't exhaustive.
6. **Detection bias** — user complaints and media drive 47% of incidents; harms to communities with less media access are underrepresented.
